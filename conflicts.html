Conflicts

Conflicts occur when the source code pulled from online repository is different from what you already had in your personal computer, and this difference appears in a similar file or part of the code. If conflicts occur, other commands cannot be performed by the users.

Solutions

In this situation, the system will notify users and give them the options to either use the code that they are working on (locally) or use the pulled code (globally). They need to choose one option for each conflict, then the file needs to be added and commited again (notes should be left). Once all conflicts are solved, they can start working normally on the project.
