Basic Git Commands

Git config

[Git config] is used to change the configuration of the project and this can be performed locally (personal devices) and globally (online repository).

Git add

[Git add "file name"] is used when the developers want to add files to the local repositories.
[Git add .] is used to add all the files, so users do not have to add single file seperately.

Git commit

[Git commit] is one of the most popular command, it is used to mark the progress of the project. For instance, once you had done working with a specific part of the project, you should use Git commit to mark that you had finished that part. All changes will be saved in the local repository, and you can return to this checkpoint at any time. Every time developers commit the code, they should always leave a message for others to know what changes they had made. [git commit -m “write comments”]

Git: Push and Pull

[Git push] is used to upload your codes to the online repositories. This should be performed when you had already committed the code.

[Git pull] is used to pull the source code from the online repository to your personal computers so you can start working with it.
